# Work Review v1.0.53

**Release date**: 2026-06-28

## New

- **Delete activity records**: the timeline can now delete activities in four granularities — by single record / by date / by time range / by app — and removes the associated screenshots and OCR text. Affected dates' daily-report and hourly-summary caches are invalidated automatically so stats and reports stay consistent (regenerated on next view).
- **UI visual styles A/B/C**: three interface styles — Quiet Pro / Soft Layers / Compact Data — each with a preview, persisted via the `ui_visual_style` config (Beta).

## Fixes

- **Chinese leaking in English mode** (#111): hardcoded UI strings that bypassed i18n (window buttons, AI test errors, avatar aria-labels, chart legends, …) were extracted into three-locale keys and aligned; the assistant's starter prompts now switch by locale (including dynamic data translation); a new `I18nEnglishLeak` regression test guards against CJK in title/aria-label/placeholder literals and keeps the three locales' keys aligned.
- **Custom categories lost in the report's time allocation** (#109): `normalize_category_key` used to fold any non-built-in category into "other", so custom categories never showed up; aggregation now keeps the custom key.
- **Timeline "period summary" button jumped to today** (#110): it now carries the viewed date; also fixed reading `?date` from `location.hash` (a latent bug under the hash router where `location.search` is empty; also affected avatar continue).
- **Linux KDE Plasma Wayland startup crash** (#113): upstream webkit2gtk/GTK protocol error on Wayland (`Error 71 dispatching to Wayland display`, Tauri #10702); we now inject `WEBKIT_DISABLE_DMABUF_RENDERER=1` before GTK init. README documents the issue and the `GDK_BACKEND=x11` fallback.
- **Responsive layout**: daily-report Markdown tables scroll horizontally, long URLs/text wrap instead of breaking the column; node-gateway device form and overview browser cards truncate correctly on narrow windows.
- **Dead updater mirrors**: the original `ghproxy.cn` + `ghp.ci` returned HTTP 000; replaced with `gh-proxy.cn` + `gh-proxy.com` (verified HTTP 200).
- **Silent crashes**: a panic hook is installed at startup that writes version/message/location/backtrace to `<data_dir>/crashes/crash-<ts>.log` (crashes used to exit with no record).
- **AiTransform model binding** (previously misrenamed to `_model`) restored.
- **Custom category names mistranslated in other locales**: custom (non-built-in) categories now keep the user-defined name instead of being overwritten by `translateCategoryLabel`; built-in categories still translate. Affects category names in the timeline, overview, and hourly chart.
- **Linux build broken** (leftover from the `commands.rs` split): after `avatar.rs` moved into the `commands/` subdirectory, its `include_str!` relative paths were one level too shallow (`../../` -> `../../../`); `system.rs` called `avatar.rs`'s private `gnome_avatar_extension_*` helpers without a `use` statement, and `avatar.rs` was missing a `PathBuf` import. Linux CI compiles again.

## Improvements

- **Desktop pet GPU usage**: removed `will-change` on `.scene-svg` (it forced the whole scene into a permanent GPU compositing layer) and reduced the drop-shadow blur radius 18→12px (visually near-identical, less per-frame filter cost).
- **Locale switcher UX**: the active language is now marked with a darker background (checkmark removed); READMEs bold their own language and no longer self-link, and "中文" became "简体中文" for clarity.
- **Code quality**: removed ~981 lines of dead code (16 unused functions + test helpers); deduped 6 privacy-filter functions into a single source in `core/privacy.rs`; split `commands.rs` (8517 lines) into 14 per-domain submodules (behavior unchanged, registration paths unchanged); workspace at 0 warnings.
- **Hourly chart duration localized**: axis ticks and legend durations now use `formatDurationLocalized` to follow the current language (previously hardcoded `Xh/Xm/Xs`).
- **README interface preview expanded**: all three READMEs add a workflow GIF and per-surface screenshots (overview / timeline / timeline detail / daily report / assistant / integrations / hourly summary / settings pages); plus a capture script and guard tests.

## macOS install note

On first launch you may see a "damaged" warning. Run:

```bash
sudo xattr -rd com.apple.quarantine '/Applications/Work Review.app'
```
