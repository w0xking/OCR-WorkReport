# Work Review v1.0.51

**發布日期**：2026-06-18

## 修復

- **活動採集誤判（根因）**：`normalize_electron_app_name` 的瀏覽器標題推斷原本對所有進程生效、缺少守衛。當視窗標題含瀏覽器關鍵詞（如 PyCharm 開啟 `test_edge_cloud.py`，檔名含 "edge"）時，明確進程名被強制改寫成瀏覽器名（PyCharm → Microsoft Edge），導致 `app_name` 與 `executable_path` 不一致——這正是「瀏覽器活動卻顯示編譯器圖示」的源頭。現僅在 Electron / Helper 類通用進程上才用標題推斷，明確進程名直接保留。
- **Safari/WebKit 內部 helper 被誤識別為獨立瀏覽器**：`com.apple.SafariPlatformSupport.Helper`、`com.apple.WebKit.Networking` 等 Safari/WebKit 的瞬態內部 XPC service 被採集為獨立 app，且 bundle id 含 "safari" 被 `is_browser_app` 判為瀏覽器，於是以原始 bundle id 作為獨立「瀏覽器」條目顯示。`normalize_display_app_name` 現將這類內部 helper 統一歸併到 Safari。
- **應用圖示錯配（顯示層）**：當 `executable_path` 與 `app_name` 不一致時，不再錯誤顯示編譯器圖示。`executable_path` 解析出的 bundle 現需通過名稱匹配校驗（`macos_score_app_bundle_name > 0`）才獲最高優先級，不匹配時回退名稱評分兜底。
- **圖示快取陳舊**：macOS 磁碟快取目錄與前端 localStorage 同步升級到 v2，失效歷史髒快取（曾因上述錯配把錯誤的編譯器圖示持久化），確保修復對存量使用者立即生效。
- **基礎模板現完整覆蓋工作查詢**：無模型模式下，含時間詞（今天/本週/最近…）或工作詞（做了什麼/總結/效率/復盤…）的查詢都走統計模板，不再漏到「無法分析」兜底；僅明顯非工作領域（天氣/股票…）才兜底。
- **缺少 `ask.emptyResponse` i18n**：Ask.svelte 引用但三語皆無此 key（曾 fallback 到字面 key）——現補齊三語。

## 優化

- **工作助手路由簡化為「相信模型」**：移除 `route_query` 中複雜意圖 / 時間段 / 工作關鍵詞等硬編碼分類規則。有模型時，所有問題統一交給 Agent，由模型自行判斷問題是否工作相關、是否呼叫工作記錄工具。非工作問題（如「今天天氣怎麼樣」）不再因含「今天」被誤判為工作查詢而返回工作統計；無模型時走模板兜底。
- **助手介面優化**：品牌色 indigo 一致化（發送按鈕漸變、使用者氣泡、步驟指示器、歡迎頁光暈、入口卡片 hover）；新消息/步驟/錯誤進入動效與按鈕按下反饋；助手氣泡 padding 打架修復。

## 文檔

- README 改為英文主頁（中文版移至 `README.zh.md`）；三語補齊 Localhost API 文檔（認證 / 接口列表 / 範例）；界面預覽圖按語言對應 `Introduction_en/zh/tw` 目錄；語言切換鏈接統一為 English · 中文 · 繁體中文。
- 新增多語言 release notes 機制（參考 cc-switch）：中文主體取自 CHANGELOG，英文 / 繁體作為獨立檔案放在 `docs/release-notes/`，發布時 CI 自動在 Release body 頂部生成語言切換鏈接。
- 新增 MIT LICENSE 檔案 + `package.json` license 欄位 + README 連結（此前倉庫無 LICENSE，法律上等於「保留所有權利」，與開源意圖矛盾）。
- README 頂部 + 關於頁加「個人使用 · 資料本地」定位聲明，弱化「員工監控」誤解。

## macOS 安裝說明

首次開啟可能提示「已損壞，無法開啟」，請執行：

```bash
sudo xattr -rd com.apple.quarantine '/Applications/Work Review.app'
```
