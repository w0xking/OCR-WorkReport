# Work Review v1.0.51

**Release date**: 2026-06-18

## Fixes

- **Activity capture (root cause)**: browser-title inference in `normalize_electron_app_name` was running on every process without a guard. When a window title contained a browser keyword (e.g. PyCharm editing `test_edge_cloud.py`, whose filename contains "edge"), the explicit process name was forcibly rewritten to a browser name (PyCharm → Microsoft Edge), leaving `app_name` and `executable_path` inconsistent in the database — the actual source of "browser activity showing a compiler icon". Title inference now runs only for generic Electron / Helper processes; explicit process names are kept as-is.
- **Safari/WebKit internal helpers misclassified as standalone browsers**: transient internal XPC services like `com.apple.SafariPlatformSupport.Helper` and `com.apple.WebKit.Networking` were captured as standalone apps, and since their bundle id contains "safari" they were flagged as browsers by `is_browser_app`, surfacing as standalone "browser" entries under their raw bundle id. `normalize_display_app_name` now merges these internal helpers into Safari.
- **App icon mismatch (display layer)**: when `executable_path` conflicts with `app_name`, the wrong compiler/IDE icon is no longer shown. The bundle resolved from `executable_path` must now pass a name-match check (`macos_score_app_bundle_name > 0`) to get top priority; otherwise it falls back to name scoring.
- **Stale icon cache**: bumped the macOS disk cache directory and frontend localStorage to v2 to invalidate historically wrong icons, so the fix takes effect immediately for existing users.
- **Basic-template now covers all work queries**: in no-model mode, queries with time words (today / this week / recently…) or work words (what did I do / summary / efficiency / review…) all hit the stat template now, instead of falling through to the canned "no AI model" reply; only clearly non-work domains (weather / stocks…) fall back.
- **Missing `ask.emptyResponse` i18n**: the key was referenced in Ask.svelte but absent in all three locales (fell back to the literal key) — now added to zh / en / tw.

## Improvements

- **Assistant routing simplified to "trust the model"**: removed the hardcoded intent / time-period / work-keyword classification rules in `route_query`. With a model configured, every question now goes to the Agent, which decides on its own whether the question is work-related and whether to call the work-records tools. Non-work questions (e.g. "how's the weather today") are no longer misrouted into work statistics just because they contain "today"; without a model it falls back to the template.
- **Assistant UI refresh**: unified the brand color (indigo) — send button gradient, user bubble tint, step spinner, welcome halo, starter-card hover; added enter transitions for new messages / steps / error callouts and press feedback on buttons; fixed assistant-bubble padding being clobbered by template utilities.

## Docs

- README is now English-first (Chinese moved to `README.zh.md`); the Localhost API docs (auth / endpoints / examples) were backfilled in English and Traditional; interface preview images now point to per-language `Introduction_en/zh/tw` directories; the language switcher is unified to English · 中文 · 繁體中文.
- Added a multi-language release-notes mechanism (à la cc-switch): the Chinese body is taken from CHANGELOG, while English / Traditional versions live as standalone files under `docs/release-notes/`; CI auto-generates language-switch links at the top of the Release body on publish.
- Added an MIT LICENSE file + `package.json` license field + README links (the repo previously had no LICENSE, legally "all rights reserved" — contradicting the open-source intent).
- README top + About page now state "personal use, data stays on device" up front, to counter the "employee monitoring" misread.

## macOS install note

On first launch you may see a "damaged" warning. Run:

```bash
sudo xattr -rd com.apple.quarantine '/Applications/Work Review.app'
```
