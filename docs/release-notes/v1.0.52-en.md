# Work Review v1.0.52

**Release date**: 2026-06-23

## New

- **AI-orchestrated report blocks**: the daily report now lets the AI decide section order based on the day's work pattern. Each block can be pinned or hidden; user preferences always override AI order. The AI order is cached in config to avoid repeat LLM calls, and sections are numbered dynamically (一、二、…) regardless of order.
- **Proactive avatar**: the model now paces itself when/what to say and its mood via `next_check_minutes`; follow-up cards are trimmed; window-drag fallback kicks in while a card or bubble is shown.
- **Dynamic starter prompts**: when an AI model is configured, the work assistant generates starter prompts from your work context (new `generate_text_with_model` command).
- **Hourly activity modes**: the overview can now show the per-hour app breakdown by week / today / custom date range.
- **Windows portable build**: a portable zip is added to the release assets (#106).

## Fixes

- **Inconsistent stats (root cause)**: privacy filtering moved to the aggregation entry point (`get_daily_stats_with_segments_filtered`), so total duration / apps / categories / hourly distribution / sites now share one consistent basis — across Overview, MCP, report generation, and saved-report re-render. Replaces the previous post-aggregation filtering.
- **Hourly buckets could exceed 60 minutes**: cross-hour activities are now sliced per hour, and overlapping intervals are counted only once (assigned to the latest observation). Buckets no longer overflow.
- **Lost duration on app switch**: when the previous app has no history record, a lightweight backfill record is now created instead of dropping the `duration_delta`.
- **MCP read endpoints didn't apply privacy filtering**: timeline / memory / sessions / current_context / weekly resource now uniformly apply "ignored apps / excluded domains".
- **MCP notifications/initialized returned an invalid empty response** (#107).
- **Linux CI apt flakiness**: swapped the azure mirror back to archive.ubuntu.com (main source); `apt-get update` now has a 180s timeout + 3 retries with backoff (this once caused a 6h cancellation on v1.0.51).

## Improvements

- **Unified light & dark theme colors**: light text consolidated from 8 slate levels to 4; dark mode fully migrated to the GitHub Dark palette (`#0d1117/#161b22/#21262d/#30363d/#484f58`), `text-white`→`#e6edf3`, all shadows get `dark:` overrides; removed the settings-save-dock gradient, centered the About update banner, removed the update toast.
- **Locale switcher UX**: the language button now shows the full language name (not just ZH/EN/TW short codes), the active option gets a ✓, and the order is unified to ZH/EN/TW.
- **UI details**: refined scrollbars and card radii, hourly activity chart switched to a 24-column grid (first/last ticks no longer overflow), placeholders when there's no data, dark-mode white-inset cleanup, settings chip shape, sidebar nav active state (#108).
- **README rewrite**: intro/features rewritten across EN/ZH/TW for accuracy; added a Localhost API wording guard test; compressed the group asset.

## macOS install note

On first launch you may see a "damaged" warning. Run:

```bash
sudo xattr -rd com.apple.quarantine '/Applications/Work Review.app'
```
